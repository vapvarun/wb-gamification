## Why this is 1.0.0 (not 1.3.0)

The plugin has not been released publicly. Internal dev iterations carried version strings up to 1.2.0, but for the world this is the first ship. We reset to 1.0.0 so the marketing narrative is clean: "a fresh, complete v1 — no betas, no half-releases".

Code-version markers updated:
- `wb-gamification.php` — `WB_GAM_VERSION = '1.0.0'`
- `readme.txt` — Stable tag: 1.0.0
- `CLAUDE.md` — Plugin Overview row

> Historical audit reports under `audit/release-runs/2026-05-03/` and `audit/release-runs/2026-05-05/` reference "1.2.0" — that's the pre-release dev label, kept as-is for traceability. Every public artefact uses 1.0.0.

## What ships in 1.0.0

### Core (already built — see Scope cards on basecamp for full reference)

- Points engine (event-sourced, signed-int, rate-limited)
- Badges + bundled 37-SVG library + OpenBadges v3 credential export + OG share
- Levels, streaks, progression
- Challenges (per-user) + Community Challenges (collective)
- Leaderboards: global, top-members, cohort-rank
- Redemption Store (rewards + stock + coupons + atomic decrement)
- Member Hub, Kudos feed, Year Recap, Earning Guide
- 9 admin pages (Tier 0 — REST-driven, 0 admin_post + 0 wp_ajax)
- Setup wizard, Manual Award, Analytics
- Native integrations: BuddyPress, WooCommerce, LearnDash; defensive gating for bbPress / Elementor
- 51 REST endpoints + OpenAPI spec + API Keys + HMAC-signed Webhooks with retry

## Build order

Revised: with #2 + #4 reduced to XS (polish on existing infra), the work compresses to ~2 sprints.

1. **Sprint 1** — Multiple point types (#1, L) + Transactional emails (#2, XS) + Login streak (#4, XS) in parallel. #1 is the longest; #2 + #4 are quick wins that unblock #5 and ship visible features fast.
2. **Sprint 2** — Public profile pages (#3, S) + Front-end submission (#5, M) in parallel; #5 depends on email pipeline from #2 being complete.

Each card moves: Figuring it out → In progress → Ready for Testing → Testing → Done. Spawn a matching QA card in **Ready for Testing** at code-complete (mirror the v1.2-era QA card pattern).

## Out of scope for 1.0.0 (deferred to 1.1.0)

The 5 high-value follow-ups go to v1.1 (see `plan/v1.1-release-plan.md`):
- Referral system
- Time-bound campaigns + multipliers
- Slack / Discord native integration template
- Web push notifications
- Visual rule builder

These are nice-to-have but not deal-blockers. Bundling them with v1.0.0 risks scope creep on a release whose primary job is "match the competitor checkbox list, then ship".
