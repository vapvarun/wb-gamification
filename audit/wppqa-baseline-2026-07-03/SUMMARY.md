## Excluded: vendored third-party libraries under `libs/`

The audit tool scans the entire shipped tree, including bundled upstream libraries we do
not patch. All high-severity `plugin-dev-rules` findings and 3 of 5 `enum-consistency`
findings originate here and are **out of scope** (upstream-maintained, ship as-is):

- `libs/woocommerce/action-scheduler/**` — WooCommerce Action Scheduler (vendored in the 1.6.1 merge). Findings: `$_POST`/`$_GET` iteration + nonce-without-cap in `ActionScheduler_Abstract_ListTable` and `WP_Async_Request` — by-design in AS's admin list-table, gated by WP screen caps. Used on millions of sites.
- `libs/easy-digital-downloads/edd-sl-sdk/**` — EDD Software Licensing SDK. Finding: nonce-without-cap in `License.php` licensing flow — upstream SDK code.

Root cause of the freshness churn: these trees were added to the release payload in 1.6.1,
so a naive full-tree scan now trips on vendor code that the 2026-05-27 baseline never saw.
Recommended tool-side fix (not blocking 1.6.2): scope the audit to exclude `libs/`,
consistent with the onboard Phase 1.5 bundled-dep exclusion rule.

## Plugin-own enum-consistency findings — verified FALSE POSITIVES

The heuristic groups by bare variable name and cannot distinguish subsystems:

1. **"event"** — `src/Admin/WebhooksAdminPage.php:344` (webhook event types:
   `badge_earned, challenge_completed, kudos_given, level_changed, points_awarded, streak_milestone`)
   vs `src/CLI/EmailCommand.php:58` (email trigger types:
   `badge_earned, challenge_completed, leaderboard_nudge, level_up, redemption, weekly_recap`).
   Two distinct subsystems (outbound webhooks vs email digests) that legitimately carry
   different value sets. Not drift — merging to one canonical list would be incorrect.

2. **"reason"** — `src/Engine/NotificationBridge.php:187` (throttle reasons:
   `cooldown, daily_cap, weekly_cap`) vs `src/Engine/RedemptionEngine.php:90`
   (redemption-failure reasons: `insufficient, ledger_write_failed, out_of_stock, record_write_failed`).
   Notification-suppression reasons ≠ redemption-failure reasons. Not drift.
